    class Solution:
    def __init__(self):
        self.stack = [-1, -1]   #initialize self.stack as [-1, -1] to avoid the problem of []
        # values in stack: ')' == -1, '(' == 0 while single '(', '(' > 0 while Valid Parentheses

    def longestValidParentheses(self, s: str) -> int:
        top = 1   #the index of the top of valid value in bucket
        for char in s:
            if char == '(':
                top += 1
                if top >= len(self.stack):
                    self.stack.append(0)       #initial '(' = 0
                else:
                    self.stack[top] = 0      #initial '(' = 0
            else:
                if self.stack[top] != -1:    #the point is not ')'
                    if self.stack[top] == 0:
                        self.stack[top] += 2
                        while self.stack[top-1] > 0:
                            top -= 1
                            self.stack[top] += self.stack[top+1]
                    elif self.stack[top-1] == 0:
                        self.stack[top-1] += 2
                        while self.stack[top-1] > 0:
                            top -= 1
                            self.stack[top] += self.stack[top+1]
                    else:
                        top += 1
                        if top >= len(self.stack):
                            self.stack.append(-1)   #initial ')' = -1
                        else:
                            self.stack[top] = -1    #initial ')' = -1

        top += 1
        max = 0
        for i in range(top):
            if self.stack[i] > max:
                max = self.stack[i]

        return max

解題說明

⏳ 時間複雜度：O(n), n = len(s)

💾 空間複雜度：O(n), n = len(s)

簡單的stack，用最暴力的將情境二分法的if...else...邏輯，並進行適當的重構化簡。

只要有一個連串的 Valid Parentheses 在 stack 中，都是以單個大於 0 的值來儲存，而其他則以 -1 表不連串的無效 ' ) ' 或 0 表不連串的 ' ( ' 來儲存。

想法很簡單，如果是 ' ( ' 就 push(0) 進 stack，如果是 ' ) ' 就判定 stack[top] 和 stack[top-1] 來決定要push(-1) 或是將 stack[top] 增值或一連串 pop 的動作。

不過這邊不使用pop而是更改了top以保存stack中所獲的的空間使多次操作基本上都是用同一塊記憶體 ( 以 C 的 linked list 的角度來看 )，若 list 的底層操作如 C 這個函式空間複雜度將會是O(t+ls/t) , t = 調用次數, ls = len(longest s)。

內建函式庫中的函式的空間複雜度也實實的影響了我對這隻程式的空間複雜度的分析，讓我非常困惑，所以上面那一段我還是只是假設，底下寫的則選用比較保守的O(n)。

Python 在各資料結構的空間管理似乎比 C 更需要認真研究才能有效的提升效能，如以上討論還不包含Python的堆積(Heap)記憶體。
