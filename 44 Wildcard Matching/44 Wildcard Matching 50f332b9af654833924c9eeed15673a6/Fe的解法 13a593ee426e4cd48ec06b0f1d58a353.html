    class Solution:
    def isMatch(self, s: str, p: str) -> bool:

        p = re.sub("\*+", "*" , p)
        p = re.sub("\?", ".", p)
        
        sub_p = p.split('*')

        if len(sub_p) < 2:
            p = re.sub("\*+", "(.*)", p)
            if re.fullmatch(p, s) is None:
                return False
            else:
                return True

        if p[0] != '*':
            if re.match(sub_p[0], s) is None:
                return False
        if p[-1] != '*':
            if re.search(sub_p[-1]+'$', s) is None:
                return False
        
        for unit in sub_p:
            outcome = re.search(unit, s)
            if outcome is None:
                return False
            else:
                s = s[s.index(outcome[0]) + len(outcome[0]):]

        return True

解題說明

如果以容易寫和容易理解為基準，那這題就會是正規表達式的主場啦！

class Solution:
    def isMatch(self, s: str, p: str) -> bool:
        P = p;
        P = re.sub('\?', '.', P)
        P = re.sub('(\*+)',"*",P)
        P = re.sub('\*', '(.*?)', P)
        detect = re.compile(r'%s'%P)
        if detect.fullmatch(s,0,len(s)):
            return True
        else:
            return False

如果照題意用上面這幾行程式碼，只能搞定簡單的輸入，若輸入較為複雜很容易就會因為NFA過於複雜導致 Time out。

所以這邊我做的一些手腳簡化這題的時間複雜度，去除一些情況不提，我這邊講主軸。

我將 p 的 (\*+) 以 "*" 取代再將 p 的 \? 以 "." 取代，然後以 * 分割成字串列表 sub_p後做頭尾的判定，若不符合直接回傳 False。接著以sub_p中的成員循序探勘 s 中有沒有符合的單位，如果沒有就 return False。這邊有一點特別值得注意，'\*' 對應 (.*) 和 '\?' 對應 (.) ，而在循序的檢查中排除了所有的 '\*' 意味著正規表達式中只包含一般字元和 (.) ，這大大的降低了比較的複雜度。

若都沒有 return False 就執行結尾的 return True。

⏳ 時間複雜度：O(n*m), n = len(s), m = the number of state in NFA

💾 空間複雜度：O(n*k), n = len(s), k = len(re.sub("\*+", "*" , p).split('*'))

時間複雜度的部分算是低估的悲觀評估，m 在實際情況還要扣除 (.*) 的部分，所以應該會快很多。

空間複雜度的部分因為 for 中的切片導致空間複雜度如此，但實際上可以達成 O(len(p))，就是每個unit都用re.compile( )處理，不過在沒有重用的情況下這樣還滿北七的。

而且沒必要，因為用切片就可以應對絕大多數情況了，而實際執行起來也不會太差。
